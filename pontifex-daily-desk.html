Pontifex without the paper cuts.

Encrypt, decrypt, key a Solitaire deck from a daily public seed, and inspect the keystream process. This is for historical and educational use only, not modern secure encryption.

Security notice: Solitaire/Pontifex is not modern secure encryption. A deterministic public seed is not a true secret. Anyone who knows or guesses the seed rule can reproduce the key. Use a private phrase if you want the exercise to be less guessable. localStorage is not secure storage.

Reference / Help

Solitaire/Pontifex uses a 54-card deck as a state machine to produce a keystream. Plaintext letters A=1 through Z=26 are added to keystream values for encryption and subtracted for decryption.

Not modern security. Public-seed-only mode is weak. Adding a private phrase makes the key less guessable, but this remains an educational historical cipher notebook. Spaces, punctuation, digits, and emoji are stripped.

Daily public seed mode is like agreeing to use a newspaper bridge hand: both parties independently derive the same key from a known public source and date. If an attacker knows the rule, the public part is reproducible.

Deterministic seed phrases are stored in history. Private phrases are not stored unless you explicitly enable remembering them; by default only a SHA-256 fingerprint is kept. localStorage is not secure storage.
