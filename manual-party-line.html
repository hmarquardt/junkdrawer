Help / About

Manual Party Line is a backend-free v1 demo. It proves a one-to-one browser video call can be set up with WebRTC while the humans manually carry the signaling messages.

How signaling works

  • User A creates an offer and sends it privately to User B.
  • User B accepts the offer, creates an answer, and sends it back.
  • User A connects with the answer. The browsers then try to establish media paths.

Privacy and limits

  • Media capture requires explicit browser permission.
  • WebRTC media is encrypted in transit by browser WebRTC mechanisms.
  • This app has no accounts, backend, or intentional media relay.
  • Manual signaling JSON can expose network/session metadata.
  • Production systems need signaling and usually TURN for reliability.
